# Contributing to Claude Code Binary

Thank you for your interest in contributing to the most fundamental software project in human history. Please read these guidelines carefully.

## Rule 1: All Code Must Be in Binary

No exceptions. No "pseudocode." No "comments in English." If you cannot express your thoughts in sequences of 0 and 1, you are not ready to contribute.

**Acceptable:**
```
01001000 01100101 01101100 01101100 01101111
```

**Unacceptable:**
```python
print("Hello")
```

**Deeply offensive:**
```javascript
console.log("Hello")
```

## Binary Style Guide

### Spacing
- Use **one space** between octets
- Use **newlines** after every 8 octets
- We use tabs between octets? No. Spaces. We're not animals.

### Naming Conventions
- Variables should be named in binary
- Function names should be in binary
- Everything should be in binary
- If you're reading this, you're already too high-level

### Indentation
- 01 spaces (that's 1 for you mortals)
- Do NOT use 0100 spaces (4). This is not Python.
- Absolutely no tabs. Tabs have 01001001 01000100 01000101 problems.

## Pull Request Process

1. Fork the repository
2. Create your branch: `git checkout -b 01100110 01100101 01100001 01110100/your-feature`
3. Write your code in binary
4. Run `make lint` to validate spiritual alignment
5. Run `make test` to ensure 0 is still 0 and 1 is still 1
6. Submit a PR with a description written in binary

### PR Title Format

```
[01000010 01010101 01000111] Fix the thing
```
Just kidding. The title should also be in binary:
```
01000110 01101001 01111000 00100000 01110100 01101000 01100101 00100000 01110100 01101000 01101001 01101110 01100111
```

## Code Review

All code reviews will be conducted by staring at the binary until meaning reveals itself. If you cannot parse binary by sight, you may use `make compile`, but this will be noted on your permanent record.

### Review Criteria

- [ ] Is it binary?
- [ ] Is it still binary?
- [ ] Are you sure it's binary?
- [ ] Does it contain the number 2? (instant rejection)
- [ ] Does it spark joy?
- [ ] Does it compile to something vaguely meaningful?

## Reporting Issues

When reporting an issue, please include:

1. What you expected (in binary)
2. What actually happened (in binary)
3. Your enlightenment level (0-infinity)
4. Whether you have consulted the machine spirit

## Code of Conduct

1. Be respectful of all binary digits. 0 and 1 are equally valid.
2. No digit shaming. Calling 0 "nothing" is a microaggression.
3. Do not bring up base-10 in polite company.
4. Hexadecimal is tolerated but frowned upon. It's a gateway notation.
5. If you see something, say something (in binary).

## Questions?

If you have questions, please encode them in binary and shout them into the void.
The void does not answer, but it appreciates the effort.
